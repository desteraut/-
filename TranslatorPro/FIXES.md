# ИСПРАВЛЕНИЯ TranslatorPro V3

## Краткое резюме

Проект TranslatorPro был полностью переработан и исправлен. Русский перевод теперь работает корректно.

## Что было сломано

### 1. Критическая ошибка: CodeProtector уничтожал естественный язык
**Файл:** `modules/m06_code_protector.py`

**Проблема:** Паттерны CodeProtector применялись ко ВСЕМУ тексту, включая обычные диалоги. Это приводило к тому, что обычные английские слова заменялись плейсхолдерами:
- `and` → `[NTP]0001`
- `or` → `[NTP]0002`
- `in` → `[NTP]0003`
- `not` → `[NTP]0004`
- `is` → `[NTP]0005`
- `.` (в конце предложения) → `[NTP]0006`

**Результат:** Текст `You and Elke sit down in the garden.` превращался в нечитаемый набор плейсхолдеров. Argos Translate получал бессмысленный набор символов и:
- Либо возвращал оригинал без изменений
- Либо переводил частично, создавая абракадабру вроде `The three of you go over to the game room in the Назад of the bar`

**Исправление:** Разделены режимы защиты:
- `DIALOGUE_PATTERNS` — для диалогов, защищает ТОЛЬКО RenPy теги, переменные, пути, HTML
- `CODE_PATTERNS` — для кода (screen/init python), агрессивная защита ключевых слов

Пайплайн теперь передаёт `is_code=False` для диалогов и `is_code=True` только для кода.

### 2. Архитектурная ошибка: результаты перевода не передавались в генератор
**Файлы:** `main.py`, `domain/pipeline/localization_pipeline.py`

**Проблема:**
1. `_translate_thread()` переводил текст и сохранял результаты в кэш
2. `_generate_tl_thread()` читал из кэша
3. Но если перевод не удался, в кэш всё равно записывался "перевод" (оригинальный текст с PUA-символами)
4. Генератор брал эти "переводы" из кэша и записывал их в файлы

**Результат:** Все файлы в `russian/` содержали оригинальный английский текст.

**Исправление:**
- `main.py` теперь хранит `last_translation_results` после перевода
- `_generate_tl_thread()` передаёт эти результаты напрямую в генератор через `translations_data`
- Пайплайн НЕ сохраняет в кэш проваленные переводы

### 3. Отсутствие валидации перевода
**Файлы:** `domain/pipeline/localization_pipeline.py`, `infrastructure/generators/renpy_generator.py`

**Проблема:** Система никак не проверяла, действительно ли текст переведён. Для русского языка файл мог содержать 100% английский текст, и система считала это нормальным.

**Исправление:** Добавлена функция `_is_actually_translated()`:
- Проверяет, что перевод не равен оригиналу
- Для русского: требует минимум 15% кириллических символов
- Невалидные переводы не сохраняются в кэш
- Генератор фильтрует невалидные переводы при создании файлов

### 4. ArgosEngine: ложная доступность
**Файл:** `infrastructure/engines/argos_engine.py`

**Проблема:**
- `_init_model()` устанавливал `_available = True` сразу после загрузки пакета
- Но пакет мог быть загружен, а Stanza (sentence boundary detection) — нет
- При первом переводе Argos пытался скачать Stanza модели с HuggingFace
- Без интернета это приводило к падению или возврату оригинала

**Исправление:**
- Добавлен тестовый перевод (`"Hello world"`) при инициализации
- `_available` устанавливается в `True` ТОЛЬКО если тестовый перевод прошёл успешно
- Добавлена обработка ошибок Stanza
- Правильная обработка кортежей от Argos

### 5. Проблема с путями
**Файл:** `infrastructure/generators/renpy_generator.py`

**Проблема:** Генератор создавал файлы в зависимости от переданного `game_dir`. Если пользователь указал корень проекта, файлы создавались в `game/tl/russian/` (правильно). Но старые запуски создавали папку `russian/` прямо в корне проекта.

**Исправление:** Уточнена логика определения пути:
```python
if (game_dir / "game").exists() and (game_dir / "game").is_dir():
    output_dir = game_dir / "game" / "tl" / self.language_folder
else:
    output_dir = game_dir / "tl" / self.language_folder
```

### 6. Дублирование защиты плейсхолдеров (PUA)
**Файл:** `domain/pipeline/localization_pipeline.py`

**Проблема:** Пайплайн заменял плейсхолдеры на Unicode PUA символы (U+E000) перед переводом. Argos мог удалять или портить эти символы. После перевода восстановление не срабатывало.

**Исправление:** Убрана Unicode PUA защита из пайплайна. ArgosEngine сам защищает плейсхолдеры через `@@NTP1@@` маркеры, которые более надёжны.

### 7. Пустой common.rpy при отсутствии интерфейсных строк
**Файл:** `infrastructure/generators/renpy_generator.py`

**Проблема:** Если все строки имели тип `dialogue` или `menu`, `common.rpy` создавался пустым или с минимумом контента.

**Исправление:** Меню теперь корректно направляются в `common.rpy` как строки интерфейса, если они имеют тип `menu`.

## Как теперь работает перевод

### Диалоги
```
Исходный: mc "You and Elke sit down in the garden."
↓
CodeProtector (is_code=False): теги/переменные → [NTP], но and/in/. НЕ трогаются
↓
ProtectionManager: [var] → ###PH_###
↓
Glossary: применяются термины
↓
ArgosEngine: [NTP] → @@NTP@@, перевод, @@NTP@@ → [NTP]
↓
Восстановление: ###PH_###, [NTP], глоссарий
↓
PostProcessor: кавычки → ёлочки, ... → …
↓
Валидация: содержит ли кириллицу? (мин. 15%)
↓
Если НЕТ → возвращаем оригинал, НЕ сохраняем в кэш
Если ДА → сохраняем в кэш
↓
Генератор: берёт из кэша/памяти, фильтрует невалидные, создаёт .rpy
```

### Код (screen, init python)
```
Исходный: textbutton "Back" action Function(renpy.music.play, "audio/back.ogg")
↓
CodeProtector (is_code=True): textbutton, action, Function, renpy.music.play, пути → [NTP]
↓
... (остальные шаги аналогично)
```

## Примечания

1. **Кэш:** Кэш сохраняется между запусками. Если вы изменили глоссарий или обновили Argos — очистите кэш через GUI или удалите `.db` файлы.

2. **Глоссарий:** Проверьте `glossary.py` — в нём встроен базовый глоссарий. Добавьте свои термины для конкретной игры.

3. **Шрифты:** Для русского языка рекомендуется использовать шрифт с поддержкой кириллицы (например, `Noto Sans` или `Liberation Sans`).

4. **Ручная проверка:** Даже с исправлениями автоматический перевод требует ручной вычитки. Используйте `Check Translation Quality` в GUI.
